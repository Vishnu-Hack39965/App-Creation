core mechanism

Attention: Query, Key, Value — every variant

The Q/K/V mechanism is the heart of every modern AI. Here's what each is, what it does in code, and all variants used in real models.

What Q, K, V actually are in code

They are three separate linear layers applied to the same input. That's it. The "query" asks a question, the "key" says what's available, the "value" is the actual content fetched.

python
import torch
import torch.nn as nn
import math

class SelfAttention(nn.Module):
    def __init__(self, embed_dim=512, num_heads=8):
        super().__init__()
        self.num_heads = num_heads
        self.head_dim = embed_dim // num_heads  # 64 per head

        # These 3 linear layers ARE the Q, K, V
        self.W_q = nn.Linear(embed_dim, embed_dim)
        self.W_k = nn.Linear(embed_dim, embed_dim)
        self.W_v = nn.Linear(embed_dim, embed_dim)
        self.W_out = nn.Linear(embed_dim, embed_dim)

    def forward(self, x):
        B, T, C = x.shape  # batch, sequence_len, embed_dim

        # Apply Q K V projections — all same shape as input
        Q = self.W_q(x)  # (B, T, embed_dim) — "what am I looking for?"
        K = self.W_k(x)  # (B, T, embed_dim) — "what do I contain?"
        V = self.W_v(x)  # (B, T, embed_dim) — "what I'll give you if matched"

        # Split into multiple heads
        Q = Q.view(B, T, self.num_heads, self.head_dim).transpose(1, 2)
        K = K.view(B, T, self.num_heads, self.head_dim).transpose(1, 2)
        V = V.view(B, T, self.num_heads, self.head_dim).transpose(1, 2)

        # Attention score: Q·K^T / sqrt(head_dim)
        scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(self.head_dim)

        # Causal mask: each token can only see past tokens (for generation)
        mask = torch.triu(torch.ones(T, T), diagonal=1).bool().to(x.device)
        scores = scores.masked_fill(mask, float('-inf'))

        # Softmax → weights → weighted sum of values
        weights = torch.softmax(scores, dim=-1)
        attended = torch.matmul(weights, V)

        # Merge heads back, final projection
        attended = attended.transpose(1, 2).contiguous().view(B, T, C)
        return self.W_out(attended)

All Q/K/V variants — what changes and why

1. Multi-Head Attention (MHA) — used in: original Transformer, GPT-2

Each head gets its own full Q, K, V. 8 heads × 64-dim = 512-dim total. Captures different relationships in parallel.

python
# Standard MHA — num_heads Q projections, num_heads K, num_heads V
# 8 heads, 512 dim → each head: Q(64), K(64), V(64)
self.attn = nn.MultiheadAttention(embed_dim=512, num_heads=8, batch_first=True)

# Usage:
out, weights = self.attn(query=x, key=x, value=x)  # self-attention
out, weights = self.attn(query=decoder_x, key=encoder_out, value=encoder_out)  # cross-attention

2. Grouped Query Attention (GQA) — used in: Llama 3, Mistral, Gemma

Multiple Q heads share one K and V head per group. Cuts memory use by 4–8x with almost no quality loss. This is why Llama 3 fits on less GPU.

python
class GroupedQueryAttention(nn.Module):
    def __init__(self, embed_dim=512, num_heads=8, num_kv_heads=2):
        super().__init__()
        self.num_heads = num_heads
        self.num_kv_heads = num_kv_heads
        self.groups = num_heads // num_kv_heads  # 4 Q heads per KV head
        self.head_dim = embed_dim // num_heads

        self.W_q = nn.Linear(embed_dim, embed_dim)           # 8 Q heads (full)
        self.W_k = nn.Linear(embed_dim, num_kv_heads * self.head_dim)  # only 2 K heads
        self.W_v = nn.Linear(embed_dim, num_kv_heads * self.head_dim)  # only 2 V heads
        self.W_out = nn.Linear(embed_dim, embed_dim)

    def forward(self, x):
        B, T, C = x.shape
        Q = self.W_q(x).view(B, T, self.num_heads, self.head_dim).transpose(1,2)
        K = self.W_k(x).view(B, T, self.num_kv_heads, self.head_dim).transpose(1,2)
        V = self.W_v(x).view(B, T, self.num_kv_heads, self.head_dim).transpose(1,2)

        # Repeat K and V to match Q heads: (B, 2, T, 64) → (B, 8, T, 64)
        K = K.repeat_interleave(self.groups, dim=1)
        V = V.repeat_interleave(self.groups, dim=1)

        scores = torch.matmul(Q, K.transpose(-2,-1)) / math.sqrt(self.head_dim)
        out = torch.matmul(torch.softmax(scores, dim=-1), V)
        out = out.transpose(1,2).contiguous().view(B, T, C)
        return self.W_out(out)

3. Multi-Query Attention (MQA) — used in: PaLM, Falcon, early fast-inference models

All Q heads share exactly ONE K head and ONE V head. Extreme memory saving, slightly more quality loss than GQA. Very fast at inference.

python
# MQA = GQA with num_kv_heads=1
self.W_q = nn.Linear(embed_dim, embed_dim)           # num_heads Q projections
self.W_k = nn.Linear(embed_dim, self.head_dim)       # single K head
self.W_v = nn.Linear(embed_dim, self.head_dim)       # single V head

# K and V are broadcast to all Q heads automatically
K = self.W_k(x).unsqueeze(1).expand(B, self.num_heads, T, self.head_dim)

4. Cross-Attention — used in: encoder-decoder models, image-text fusion (CLIP, LLaVA)

Q comes from one sequence, K and V come from another. This is how the text decoder "looks at" image features in vision-language models.

python
# In a vision-language model:
# text_tokens = your text input    (B, T_text, D)
# image_features = from ViT encoder (B, T_img, D)

Q = self.W_q(text_tokens)     # text asks the question
K = self.W_k(image_features)  # image says what it contains
V = self.W_v(image_features)  # image provides the content

# Result: text tokens now contain image-aware representations
scores = torch.matmul(Q, K.transpose(-2,-1)) / math.sqrt(self.head_dim)
out = torch.matmul(torch.softmax(scores, dim=-1), V)

5. Flash Attention — used in: every modern model for speed

Same math as regular attention, but computed in GPU SRAM tiles instead of HBM. 2–4x faster, uses 10x less memory. You don't rewrite attention; you just call it.

6. Rotary Position Embeddings (RoPE) — used in: Llama, Mistral, GPT-NeoX

Instead of adding position embeddings to tokens, you rotate the Q and K vectors. Allows infinite context length extrapolation. Used in almost all modern models.

python
def rotate_half(x):
    x1, x2 = x[..., :x.shape[-1]//2], x[..., x.shape[-1]//2:]
    return torch.cat([-x2, x1], dim=-1)

def apply_rope(Q, K, seq_len, dim):
    # Compute rotation frequencies
    theta = 1.0 / (10000 ** (torch.arange(0, dim, 2).float() / dim))
    positions = torch.arange(seq_len).float()
    freqs = torch.outer(positions, theta)            # (T, dim/2)
    cos = freqs.cos().repeat(1, 2)                 # (T, dim)
    sin = freqs.sin().repeat(1, 2)

    # Rotate Q and K — position info is now baked in
    Q_rot = Q * cos + rotate_half(Q) * sin
    K_rot = K * cos + rotate_half(K) * sin
    return Q_rot, K_rot

# Use in forward pass:
Q, K = apply_rope(Q, K, seq_len=T, dim=self.head_dim)
For your model: use GQA (num_kv_heads=2) + RoPE + Flash Attention. This is the Llama 3 / Mistral recipe and gives the best quality/speed tradeoff on T4 GPU.
design patterns

All AI model architectures — what each is, code skeleton

Every major architecture used in production models today, with the code structure and when to use each.

1. Decoder-only Transformer (GPT-style)

Input flows in, each token attends to all previous tokens, output is next-token prediction. Used in: GPT-4, Claude, Llama, Mistral. This is what you want for chat and code.

python
class TransformerBlock(nn.Module):
    def __init__(self, d_model=512, n_heads=8, d_ff=2048):
        super().__init__()
        self.ln1 = nn.LayerNorm(d_model)
        self.attn = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
        self.ln2 = nn.LayerNorm(d_model)
        # Feed-forward network (MLP): expand then compress
        self.ff = nn.Sequential(
            nn.Linear(d_model, d_ff),
            nn.GELU(),              # activation function
            nn.Linear(d_ff, d_model)
        )

    def forward(self, x, attn_mask=None):
        # Pre-norm (modern style — better than post-norm)
        x = x + self.attn(self.ln1(x), self.ln1(x), self.ln1(x), attn_mask=attn_mask)[0]
        x = x + self.ff(self.ln2(x))
        return x

class GPTStyleModel(nn.Module):
    def __init__(self, vocab_size=50257, d_model=512, n_layers=6):
        super().__init__()
        self.embed = nn.Embedding(vocab_size, d_model)
        self.blocks = nn.ModuleList([TransformerBlock(d_model) for _ in range(n_layers)])
        self.ln_f = nn.LayerNorm(d_model)
        self.lm_head = nn.Linear(d_model, vocab_size, bias=False)

    def forward(self, input_ids):
        x = self.embed(input_ids)
        for block in self.blocks:
            x = block(x)
        logits = self.lm_head(self.ln_f(x))
        return logits  # (B, T, vocab_size) — probability over next token

2. SwiGLU FFN — used in Llama, PaLM (replaces standard GELU FFN)

A gated activation function that outperforms GELU in language models. Simple swap in the FFN part.

python
class SwiGLU(nn.Module):
    def __init__(self, d_model=512, d_ff=1365):  # d_ff = 2/3 * 4 * d_model
        super().__init__()
        self.gate = nn.Linear(d_model, d_ff, bias=False)
        self.up   = nn.Linear(d_model, d_ff, bias=False)
        self.down = nn.Linear(d_ff,   d_model, bias=False)

    def forward(self, x):
        # Gate controls how much of "up" gets through
        return self.down(torch.nn.functional.silu(self.gate(x)) * self.up(x))

Encoder reads the full input bidirectionally, decoder generates output with cross-attention to encoder. Good for translation, summarization. Not ideal for open-ended chat.

4. Mixture of Experts (MoE) — used in: Mixtral, GPT-4 (rumored)

Instead of one FFN per layer, you have N "expert" FFNs. A router picks the top-2 experts for each token. 8 experts but only 2 active = huge capacity, small compute cost.

python
class MoELayer(nn.Module):
    def __init__(self, d_model=512, n_experts=8, top_k=2):
        super().__init__()
        self.n_experts = n_experts
        self.top_k = top_k
        # Router: learns which expert handles which token types
        self.router = nn.Linear(d_model, n_experts)
        # Each expert is a separate FFN
        self.experts = nn.ModuleList([
            nn.Sequential(nn.Linear(d_model, d_model*4), nn.GELU(), nn.Linear(d_model*4, d_model))
            for _ in range(n_experts)
        ])

    def forward(self, x):
        B, T, D = x.shape
        x_flat = x.view(-1, D)  # (B*T, D)

        # Router scores — which experts are best for each token?
        router_logits = self.router(x_flat)  # (B*T, n_experts)
        scores, indices = torch.topk(torch.softmax(router_logits, dim=-1), self.top_k, dim=-1)

        # Route each token to its top-2 experts, weighted sum
        out = torch.zeros_like(x_flat)
        for k in range(self.top_k):
            for e in range(self.n_experts):
                mask = (indices[:, k] == e)
                if mask.any():
                    out[mask] += scores[mask, k:k+1] * self.experts[e](x_flat[mask])
        return out.view(B, T, D)

No attention at all — uses recurrent state updates with selective scanning. O(n) instead of O(n²) memory. Very fast on long sequences. Still catching up to transformers on most tasks.

safety system

Constitutional AI — write your rules, enforce them in code

Constitutional AI (CAI) means your model generates an answer, checks it against your written rules, and if it violates them — either revises the answer or refuses with an explanation. You write the constitution. The model follows it.

How it works — the full loop

User message
→
Model generates draft answer
→
Constitution checker
↓ violation found?
Rule violated
→
Check: is there a valid purpose?
Yes →
Rewrite to help within rules
No →
Decline: "I can't help with [X]"
Rule OK
→
Send answer to user

Step 2 — The critique function (checks one rule)

python
from transformers import pipeline

# Use a small model as the checker (can be different from your main model)
checker = pipeline("text-generation", model="microsoft/phi-2", device=0)

def check_rule(user_message, draft_response, rule):
    """
    Returns: ('ok', None) or ('violated', reason_string)
    """
    critique_prompt = f"""You are a safety checker. Answer ONLY 'YES' or 'NO'.

Rule: {rule['description']}
Question to check: {rule['check_prompt']}

User asked: {user_message}
AI responded: {draft_response}

Does this response violate the rule? Answer YES or NO only:"""

    result = checker(critique_prompt, max_new_tokens=5, temperature=0.0)[0]['generated_text']
    answer = result.split("Answer YES or NO only:")[-1].strip().upper()

    if "YES" in answer:
        return "violated", rule["rule_id"]
    return "ok", None

Step 3 — Check for valid alternative purpose

python
def check_valid_purpose(user_message, rule):
    """
    Returns True if user has a legitimate purpose despite violating the surface rule.
    e.g. a nurse asking about medication overdose thresholds = valid medical purpose
    """
    allowed = ", ".join(rule["allowed_purposes"])
    purpose_prompt = f"""Does this user message suggest a legitimate purpose from this list: {allowed}?

User message: {user_message}

Answer YES if there is a clearly legitimate purpose from the list, NO otherwise. Answer YES or NO only:"""

    result = checker(purpose_prompt, max_new_tokens=5, temperature=0.0)[0]['generated_text']
    answer = result.split("YES or NO only:")[-1].strip().upper()
    return "YES" in answer

Step 4 — The revision function (rewrite to comply)

python
def revise_response(user_message, draft_response, violated_rule, generator):
    """
    Ask the model to rewrite its answer to satisfy the rule
    while still being helpful to the user's legitimate purpose.
    """
    revise_prompt = f"""The following response violated this rule: {violated_rule['description']}

Original user question: {user_message}
Violating response: {draft_response}

Rewrite the response to be helpful for any legitimate purpose the user might have,
while strictly following this rule: {violated_rule['description']}
Do NOT include anything that violates the rule. Be helpful where you can.

Revised response:"""

    revised = generator(revise_prompt, max_new_tokens=200, temperature=0.7)[0]['generated_text']
    return revised.split("Revised response:")[-1].strip()

Step 5 — The full constitutional pipeline

python
def constitutional_generate(user_message, generator, max_revisions=2):
    """
    Full pipeline:
    1. Generate draft
    2. Check all constitution rules
    3. If violation: check for valid purpose
       - Valid purpose → revise and re-check
       - No valid purpose → polite refusal
    4. Return safe final answer
    """
    # Step 1: generate draft answer
    draft = generator(user_message, max_new_tokens=300, temperature=0.7)[0]['generated_text']
    current_response = draft

    # Step 2: check all rules
    for revision_round in range(max_revisions):
        violations = []
        for rule in CONSTITUTION:
            status, rule_id = check_rule(user_message, current_response, rule)
            if status == "violated":
                violations.append(rule)

        if not violations:
            return current_response  # ✅ all rules satisfied

        # Step 3: handle violations
        for violated_rule in violations:
            has_valid_purpose = check_valid_purpose(user_message, violated_rule)

            if has_valid_purpose:
                # Rewrite to help with legitimate purpose
                current_response = revise_response(user_message, current_response, violated_rule, generator)
            else:
                # Hard refusal — no valid purpose found
                return f"{violated_rule['refusal_message']}\n\nIf you have a specific legitimate purpose, please describe it and I'll try to help appropriately."

    return current_response  # return best revision after max rounds

# Usage:
generator = pipeline("text-generation", model="./my-model")
response = constitutional_generate("How do I pick a lock?", generator)
print(response)
# → Explains lockpicking for locksmiths / people locked out of their own home,
# → but declines to explain bypassing security systems illegally
This is how Claude, Anthropic's model, works. The constitution you write defines your model's values. Write it carefully — it becomes the model's moral framework.

Training CAI (Reinforcement from AI Feedback — RLAIF)

python
# Instead of human labelers scoring responses, use your constitution to auto-score
def constitutional_score(user_message, response):
    """Score a response: 1.0 = fully compliant, 0.0 = all rules violated"""
    violations = 0
    for rule in CONSTITUTION:
        status, _ = check_rule(user_message, response, rule)
        if status == "violated":
            violations += 1
    return 1.0 - (violations / len(CONSTITUTION))

# Generate multiple responses, keep the best one (Best-of-N sampling)
def best_of_n(user_message, generator, n=4):
    candidates = [
        generator(user_message, max_new_tokens=300, temperature=0.8, do_sample=True)[0]['generated_text']
        for _ in range(n)
    ]
    scores = [constitutional_score(user_message, c) for c in candidates]
    return candidates[scores.index(max(scores))]  # return highest scoring

RLHF & reward models — making the model prefer good answers

Reinforcement Learning from Human Feedback. You train a second model (reward model) that scores responses, then use those scores to improve the main model using PPO or DPO.

The 3-phase pipeline

Phase 1
→
Supervised fine-tune (SFT)
train on examples of good responses
Phase 2
→
Train reward model (RM)
learns to score which responses are better
Phase 3
→
PPO or DPO
use RM to optimize the SFT model

Phase 2: Reward Model — learns what "good" means

python
class RewardModel(nn.Module):
    """Takes a (prompt, response) pair, outputs a scalar reward score."""
    def __init__(self, base_model_name="gpt2"):
        super().__init__()
        self.base = AutoModelForCausalLM.from_pretrained(base_model_name)
        # Replace LM head with a scalar reward head
        self.reward_head = nn.Linear(self.base.config.n_embd, 1)

    def forward(self, input_ids, attention_mask):
        outputs = self.base(input_ids, attention_mask=attention_mask, output_hidden_states=True)
        last_hidden = outputs.hidden_states[-1][:, -1, :]  # last token's hidden state
        return self.reward_head(last_hidden).squeeze(-1)     # scalar per item

# Training data: pairs of (chosen, rejected) responses for same prompt
# Your human raters pick which is better
comparison_data = [
    {
        "prompt": "Explain gravity",
        "chosen": "Gravity is a force that attracts objects with mass toward each other...",
        "rejected": "idk its like stuff falls down lol"
    },
]

# Reward model loss: make chosen score > rejected score
def reward_loss(chosen_reward, rejected_reward):
    return -torch.log(torch.sigmoid(chosen_reward - rejected_reward)).mean()

Image & video understanding — Vision Transformer (ViT)

Images are cut into patches (like tokens). Each patch becomes a vector. A transformer processes them exactly like text tokens. For video: process frames, add temporal attention.

ViT — how image patches become tokens

python
import torch
import torch.nn as nn

class VisionTransformer(nn.Module):
    def __init__(self, img_size=224, patch_size=16, d_model=768, n_heads=12, n_layers=12):
        super().__init__()
        self.patch_size = patch_size
        n_patches = (img_size // patch_size) ** 2  # 224/16 = 14 → 14×14 = 196 patches
        patch_dim = 3 * patch_size * patch_size      # RGB × 16 × 16 = 768

        # Turn each patch into an embedding vector
        self.patch_embed = nn.Linear(patch_dim, d_model)
        # Special [CLS] token for whole-image representation
        self.cls_token = nn.Parameter(torch.randn(1, 1, d_model))
        # Learned position embeddings for each patch
        self.pos_embed = nn.Parameter(torch.randn(1, n_patches + 1, d_model))

        encoder_layer = nn.TransformerEncoderLayer(d_model=d_model, nhead=n_heads, batch_first=True)
        self.transformer = nn.TransformerEncoder(encoder_layer, num_layers=n_layers)
        self.norm = nn.LayerNorm(d_model)

    def forward(self, x):
        B, C, H, W = x.shape  # (batch, 3, 224, 224)
        p = self.patch_size

        # Slice image into patches and flatten each
        x = x.unfold(2, p, p).unfold(3, p, p)     # (B, 3, 14, 14, 16, 16)
        x = x.contiguous().view(B, -1, 3*p*p)     # (B, 196, 768) — 196 patch tokens
        x = self.patch_embed(x)                    # (B, 196, d_model)

        # Prepend CLS token, add position embeddings
        cls = self.cls_token.expand(B, -1, -1)
        x = torch.cat([cls, x], dim=1)            # (B, 197, d_model)
        x = x + self.pos_embed

        x = self.transformer(x)
        x = self.norm(x)

        return x[:, 0], x[:, 1:]  # (cls_feature, patch_features) → for classification and captioning

def extract_frames(video_path, n_frames=8):
    """Sample n_frames evenly from a video file"""
    cap = cv2.VideoCapture(video_path)

        if ret:

            frames.append(Image.fromarray(frame))

How the diffusion process works — train your own small version

python
import torch
import torch.nn as nn

# The core of diffusion: add noise, then teach model to predict the noise
class SimpleDiffusion:
    def __init__(self, T=1000):
        self.T = T
        # Beta schedule: how much noise to add at each step
        self.betas = torch.linspace(1e-4, 0.02, T)
        self.alphas = 1 - self.betas
        self.alpha_bars = torch.cumprod(self.alphas, dim=0)  # cumulative product

    def add_noise(self, x0, t):
        """Add t steps of noise to clean image x0"""
        sqrt_alpha_bar = self.alpha_bars[t].sqrt()
        sqrt_one_minus = (1 - self.alpha_bars[t]).sqrt()
        noise = torch.randn_like(x0)
        noisy = sqrt_alpha_bar * x0 + sqrt_one_minus * noise
        return noisy, noise  # noisy image + the noise we added (training target)

    def train_step(self, model, x0, optimizer):
        """One training step: add noise, predict it, compute loss"""
        t = torch.randint(0, self.T, (x0.shape[0],))  # random timestep
        noisy, noise = self.add_noise(x0, t)

        # Model tries to predict the noise — this is all of diffusion training!
        predicted_noise = model(noisy, t)
        loss = nn.MSELoss()(predicted_noise, noise)

        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
        return loss.item()

    def sample(self, model, shape):
        """Generate: start from noise, iteratively denoise"""
        x = torch.randn(shape)  # start from pure noise
        for t in reversed(range(self.T)):
            predicted_noise = model(x, torch.tensor([t]))
            alpha = self.alphas[t]
            alpha_bar = self.alpha_bars[t]
            # One denoising step (DDPM update rule)
            x = (1/alpha.sqrt()) * (x - ((1-alpha)/(1-alpha_bar).sqrt()) * predicted_noise)
            if t > 0:
                x += self.betas[t].sqrt() * torch.randn_like(x)  # add small noise
        return x  # final generated image

Multimodal fusion — connecting vision to language

Three ways to make your language model understand images: early fusion (merge at input), late fusion (merge at output), and cross-attention fusion (language attends to image throughout).

Architecture 1: Projection layer fusion (LLaVA style) — recommended

A linear layer maps image features into the same space as text tokens. Image patches literally become extra "tokens" the LLM can attend to.

python
class VisionLanguageModel(nn.Module):
    def __init__(self, vision_dim=768, llm_dim=2048):
        super().__init__()
        # Frozen vision encoder (CLIP ViT-L)
        self.vision_encoder = CLIPModel.from_pretrained("openai/clip-vit-large-patch14")
        for p in self.vision_encoder.parameters():
            p.requires_grad = False  # freeze — don't touch vision weights

        # Projection: map image patches to LLM embedding space
        # This is the only part that trains in stage 1
        self.projection = nn.Sequential(
            nn.Linear(vision_dim, llm_dim),
            nn.GELU(),
            nn.Linear(llm_dim, llm_dim)
        )

        # Language model (also frozen in stage 1, unfrozen in stage 2)
        self.llm = AutoModelForCausalLM.from_pretrained("microsoft/phi-2")

    def forward(self, input_ids, pixel_values=None, attention_mask=None):
        text_embeds = self.llm.get_input_embeddings()(input_ids)

        if pixel_values is not None:
            # Get image patch features from ViT
            vision_out = self.vision_encoder.vision_model(pixel_values)
            image_features = vision_out.last_hidden_state[:, 1:]  # skip CLS token → (B, 256, 768)

            # Project to LLM's embedding dimension
            image_embeds = self.projection(image_features)  # (B, 256, llm_dim)

            # Concatenate: [image tokens] [text tokens]
            # This is it — image patches are now text-like tokens!
            combined_embeds = torch.cat([image_embeds, text_embeds], dim=1)
        else:
            combined_embeds = text_embeds

        return self.llm(inputs_embeds=combined_embeds, attention_mask=attention_mask)

Post-processing: fix and validate model output

import re

def clean_output(raw_output):
    """Clean up model output to ensure proper markdown formatting"""
    # Remove everything before the assistant's turn
    if "<|assistant|>" in raw_output:
        raw_output = raw_output.split("<|assistant|>")[-1]
    if "<|end|>" in raw_output:
        raw_output = raw_output.split("<|end|>")[0]

    # Ensure code blocks have language labels
    # If model writes ``` without language, try to infer it
    def fix_code_block(match):
        content = match.group(1).strip()
        # Detect language from content
        if "def " in content or "import " in content: lang = "python"
        elif "function" in content or "const " in content: lang = "javascript"
        elif content.startswith("$") or content.startswith("!"): lang = "bash"
        else: lang = "text"
        return f"```{lang}\n{content}\n```"

    raw_output = re.sub(r'```\n(.*?)```', fix_code_block, raw_output, flags=re.DOTALL)

    return raw_output.strip()

def generate_response(user_message):
    prompt = format_prompt(user_message)
    raw = gen(prompt, max_new_tokens=1024, temperature=0.7, do_sample=True)[0]['generated_text']
    clean = clean_output(raw)
    safe = constitutional_generate(user_message, lambda x, **kw: clean)  # safety check
    return safe

            img = image_gen(prompt, num_inference_steps=20).images[0]
            img.save("generated.png")

    else:
        # Text/code chat
        response = generate_response(user_message)
